# スニペットの追加・編集方法

このテンプレートでは、VS Code の機能を利用して LaTeX 向けのスニペット（入力補完）を管理しています。
プロジェクト固有の定型文や、よく使う数式・コマンドがある場合は、スニペットを追加・編集することで執筆を大幅に効率化できます。

## 編集するファイル

スニペットは以下のファイルで一元管理されています。

- `.vscode/latex.code-snippets`

このファイルは JSON 形式（コメント付き）で記述されており、保存すると即座に VS Code 上で反映されます。エディタの再起動などは不要です。

## スニペットの基本構造

新しいスニペットを追加する際は、ファイルの `{ ... }` の中に以下のような形式で追記します。
（※各スニペットの間は `,` （カンマ）で区切る必要があります）

```json
"スニペットの名前 (管理用)": {
    "prefix": "トリガーとなる文字列",
    "body": [
        "展開される内容の1行目",
        "展開される内容の2行目"
    ],
    "description": "補完時のリストに表示される説明"
}
```

### 記述のポイント

1. **`prefix`**: ここで指定した文字をエディタ上で入力し `Tab` を押すと展開されます。（例: `mycmd`）
2. **`body`**: 展開される中身です。改行を含む場合は、行ごとに `"` で囲んで `,`（カンマ）で区切った配列の形で記述します。
3. **エスケープ文字**: LaTeX のコマンドで使う `\`（バックスラッシュ）や、`"`（ダブルクォーテーション）を記述したい場合は、`\\` や `\"` のようにエスケープする必要があります。
   - 例: `\begin{...}` と書きたい場合 → `"\\begin{...}"`

## カーソル移動（タブストップ）の制御

展開後に `Tab` キーで入力箇所を順番に移動させることができます。これをタブストップと呼びます。

- **`$1`, `$2`, `$3`...**: 展開直後に最初のカーソルが `$1` に来ます。`Tab` を押すたびに番号順に移動します。
- **`${1:プレースホルダー}`**: デフォルトのテキストを入れておき、選択状態にすることができます。そのまま入力すれば上書きされます。（例: `${1:caption}`）
- **`$0`**: 一連のタブストップを終えた後、最後にカーソルが置かれる位置です。これを指定しない場合は、スニペットの末尾が最終位置になります。

### 具体例

例えば、「自分用の特殊な図の挿入フォーマット」を作りたい場合は、以下のように追加します。

```json
"My Custom Figure": {
    "prefix": "myfig",
    "body": [
        "\\begin{figure}[H]",
        "    \\centering",
        "    \\includegraphics[width=${1:0.8}\\linewidth]{${2:image-path}}",
        "    \\caption{${3:caption}}",
        "    \\label{fig:${4:label}}",
        "\\end{figure}",
        "$0"
    ],
    "description": "カスタマイズされた図の挿入"
}
```

これを保存して `.tex` ファイル上で `myfig` と入力して展開すると、`0.8` が選択された状態で展開され、`Tab` を押すたびに画像のパス、キャプション、ラベルの部分へと順にカーソルが移動し、最後に図の直下にカーソルが置かれます。
